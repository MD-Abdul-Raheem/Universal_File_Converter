1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
The Universal File Converter (UFC) is a high-performance, browser-based utility
application designed to democratize file transformation. It solves the problem of
format incompatibility by providing a unified, secure, and intelligent interface
for converting documents, data spreadsheets, presentations, and images.

Unlike traditional converters that rely heavily on server-side queuing systems,
UFC utilizes a "Thick Client" architecture enhanced by Generative AI. This
approach allows for instant processing, strictly typed binary generation, and
superior data privacy.

2. TECHNOLOGY STACK
--------------------------------------------------------------------------------
The application is built on a modern, type-safe stack designed for stability and
scalability.

A. Core Frameworks
   •  Language:       TypeScript (Strict Mode)
   •  Frontend Lib:   React 19 (Functional Components + Hooks)
   •  Build Tool:     Vite (ESModule serving)
   •  Styling:        Tailwind CSS (Utility-first, Custom Animations)

B. Intelligence Layer
   •  AI Model:       Google Gemini 1.5 Flash (via @google/genai SDK)
   •  Role:           Semantic understanding of unstructured data (PDF/DOCX)
                      to facilitate complex format reconstruction.

C. Binary Processing Engines (WebAssembly/JS)
   •  SheetJS (xlsx): Deterministic parsing for Excel/CSV/JSON.
   •  PptxGenJS:      Client-side generation of native PowerPoint slides.
   •  Packer (docx):  Generation of OOXML Word documents.
   •  jsPDF:          Programmatic PDF creation.
   •  Mammoth:        DOCX to HTML extraction.

3. OPERATIONAL MECHANICS (HOW IT WORKS)
--------------------------------------------------------------------------------
The system operates on a Hybrid Processing Pipeline that routes tasks based on
complexity and data structure.

PHASE 1: INGESTION & ANALYSIS
   The user drags a file into the "Data Ingestion Port". The system instantly
   analyzes the MIME type and file signature to determine the optimal processing
   route.

PHASE 2: DYNAMIC ROUTING
   The `conversionService.ts` module acts as the traffic controller:

   [ROUTE A] Deterministic (100% Precision)
   Used for: CSV, JSON, Excel
   Mechanism: The file is parsed into a raw JavaScript object or 2D array.
   Mathematical logic transforms the structure (e.g., pivoting a JSON object
   into CSV rows) and writes the result directly to a binary Blob.

   [ROUTE B] Generative Reconstruction (Semantic)
   Used for: PDF -> Word, Image -> Text, PDF -> PPT
   Mechanism: The file content is sent to the Gemini 1.5 Flash model with a
   strict "System Instruction". The AI does not simply "convert"; it extracts
   content and restructures it into a standard JSON schema (e.g., extracting
   slide titles and bullet points). The frontend then uses this JSON to
   programmatically build a brand new file using libraries like PptxGenJS.

PHASE 3: BINARY ASSEMBLY
   The resulting data stream is assembled into a strictly typed Blob (Binary
   Large Object) with the correct MIME headers, ensuring the downloaded file
   is valid and native to the target application (Microsoft Office, Acrobat, etc.).

4. KEY ADVANTAGES
--------------------------------------------------------------------------------
1.  Data Sovereignty & Privacy
    For data-heavy formats (CSV/XLSX), processing happens entirely within the
    browser's memory. No sensitive financial or personal data leaves the
    client machine.

2.  Zero-Hallucination Data Conversion
    By bypassing AI for structured data (using SheetJS instead), we guarantee
    that numbers, dates, and formulas are preserved exactly, eliminating the
    risk of AI "hallucination."

3.  True Native File Generation
    UFC creates real `.pptx` and `.docx` files with valid XML internal structures,
    not just text files renamed with a different extension.

4.  Midnight Aurora Interface
    A professional, high-contrast "Dark Mode" UI designed for extended use,
    featuring reduced eye strain, clear typographic hierarchy (Inter/JetBrains Mono),
    and immediate visual feedback via a HUD-style dashboard.

5.  Ephemeral Architecture
    No database requirements. No storage buckets. The application is purely
    functional and stateless, reducing hosting costs and liability.
